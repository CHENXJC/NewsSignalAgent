# Screenshot Guide

Use **Hybrid Demo + Live** for the richest showcase story. Refresh the six-source pilot shortly before capture, then confirm no account details, local usernames, tokens, or terminal windows are visible.

Recommended setup:

- Browser viewport: 1440 px wide
- Browser zoom: 90% or 100%
- Save to: `portfolio/showcase_screenshots/`
- Format: PNG
- Keep Demo Data available as the stable fallback if live sources change

## Required screenshot plan

1. `01_home_overview.png` - Hero, feature cards, mode guidance, and top metrics.
2. `02_global_signal_overview.png` - Top signals table and several signal cards.
3. `03_china_policy_signal.png` - China policy/demo signal section; make the demo context visible.
4. `04_cross_country_comparison.png` - Topic-region framing matrix and comparison chart.
5. `05_narrative_gap_detector.png` - Top gaps, score metrics, and one expanded neutral explanation.
6. `06_fetch_status_live_pilot.png` - Pilot fetch metrics and health table after a bounded health check.
7. `07_source_architecture.png` - Source groups, access modes, verification status, and capture policy.
8. `08_export_report_preview.png` - Sanitized Markdown report preview or successful local export confirmation.

## Capture checklist

- Use only demo data or short permitted feed metadata.
- Do not show `.env`, browser accounts, personal paths, API keys, logs, or private notes.
- Keep disclaimers and neutral framing language visible where practical.
- Review every image at full size before publishing.
- Do not fabricate screenshots; if a screen cannot be captured safely, leave its placeholder for NEWS-007.
